# Retro Protocol

Use this protocol after the content is published.

## Timing

Recommended checkpoints:

- T+1
- T+3
- T+7

Use platform-specific timing if the platform has a known longer or shorter distribution cycle.

## Required Fields

Record:

- publish timestamp
- platform
- account or column
- content form
- primary metric
- impressions, reads, or plays
- likes
- comments
- saves
- shares
- follower growth
- leads or conversions when relevant
- paid traffic status
- abnormal factors
- actual bucket
- whether raw bucket hit
- whether adjusted bucket hit
- likely error source

## Error Analysis

Classify the main miss source:

- rubric definition issue
- weight issue
- package / cover / title issue
- platform timing issue
- sample baseline issue
- abnormal distribution
- business metric mismatch

Do not change the rubric because of one miss. Use repeated directional error as the threshold for changing system rules.
